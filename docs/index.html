Características

Todo en un solo plugin

Diseñado para servidores de Minecraft modernos que necesitan un sistema de recompensas flexible y fácil de administrar.

🎁

GUI Visual Completa

Inventario completamente personalizable: título, tamaño, slot de cada recompensa, materiales por estado y botones de acción configurables vía YAML.

⏱

Cooldown por Recompensa

Cada recompensa tiene su propio cooldown independiente. Soporte para valores como 86400 (24h). El tiempo restante se muestra en el lore en tiempo real.

🔒

Permisos por Recompensa

Cada recompensa requiere un permiso específico (frewards.<id>). Si el jugador no lo tiene, ve el estado Bloqueada con lore personalizado.

🎨

Colores HEX Completos

Soporte total para colores hexadecimales (&#RRGGBB), degradados por letra y códigos legacy en todos los campos: nombres, lore, títulos y subtítulos.

📦

Ítems Físicos + Comandos

Al reclamar, el plugin puede entregar ítems configurados por el editor GUI y ejecutar múltiples comandos de consola con soporte para el placeholder %player%.

✏️

Editor GUI en Vivo

Gestiona todas las recompensas desde el juego sin tocar ningún archivo YAML. Edita ítems, comandos, cooldown y nombre visual directamente in-game.

🔔

Notificación al Conectar

Al entrar al servidor, el jugador recibe automáticamente un mensaje indicando cuántas recompensas tiene pendientes de reclamar.

🌐

Multi-idioma (ES / EN)

Sistema de idiomas con archivos lang_es.yml y lang_en.yml intercambiables desde config.yml sin reiniciar el servidor.

🎬

Título + Sonido al Reclamar

Cada recompensa puede mostrar un título/subtítulo en pantalla y reproducir un sonido personalizado al ser reclamada exitosamente.

🔄

Reload sin Reiniciar

Recarga toda la configuración en caliente con /frewards reload. Los cambios del editor GUI se aplican inmediatamente sin interrupciones.

Los 3 estados de una recompensa

Cada recompensa se muestra diferente según la situación del jugador.

✔ Disponible

El jugador tiene el permiso y el cooldown ha expirado. Puede hacer clic para reclamar la recompensa inmediatamente.

⏳ En Progreso

El jugador ya reclamó la recompensa y está esperando que expire el cooldown. El lore muestra el tiempo restante con el placeholder %cooldown_id%.

✘ Bloqueada

El jugador no tiene el permiso requerido para esta recompensa. El lore muestra qué permiso o rango necesita para desbloquearla.

Comandos

Comandos del plugin

Comandos para jugadores y administradores con autocompletado integrado.

Comando Descripción Permiso
/frewards Abre el menú de recompensas diarias. frewards.use
/rewards Alias de /frewards. Abre el mismo menú. frewards.use
/recompensas Alias de /frewards en español. frewards.use
/frewards help Muestra el panel de ayuda con todos los subcomandos disponibles. frewards.use
Comando Descripción Permiso
/frewards editor Abre el selector de recompensas del editor GUI. Muestra todas las recompensas configuradas. frewards.editor
/frewards editor <id> Abre directamente el editor de una recompensa específica por su ID (ej: ros, venus). frewards.editor
/frewards reload Recarga rewardsgui.yml, messages.yml, editor.yml y el idioma activo sin reiniciar. frewards.reload
/frewards reset <jugador> Reinicia todos los cooldowns de un jugador para que pueda reclamar todas las recompensas de nuevo. frewards.admin
/frewardsreload Comando rápido para recargar el plugin sin argumentos extra. frewards.reload
/frewardsreset <jugador> Comando rápido para reiniciar cooldowns de un jugador. frewards.admin
💡
Autocompletado integrado

El plugin incluye un TabCompleter que sugiere subcomandos e IDs de recompensas automáticamente al escribir en el chat.

Permisos

Sistema de permisos

Control total sobre quién puede hacer qué, tanto para jugadores como para administradores y por rango de recompensa.

— Permisos generales

frewards.use
Permite abrir el menú de recompensas con /frewards, /rewards y /recompensas.
default: todos los jugadores
frewards.editor
Permite abrir el editor GUI con /frewards editor y /frewards editor <id>.
default: OP
frewards.reload
Permite recargar la configuración del plugin con /frewards reload.
default: OP
frewards.admin
Permite reiniciar cooldowns de jugadores con /frewards reset y /frewardsreset.
default: OP

— Permisos por recompensa

frewards.<reward_id>
Cada recompensa tiene su propio permiso basado en su ID. Sin él, el jugador ve el estado Bloqueada.
default: false (configurable)
frewards.ros
frewards.venus
frewards.apolo…
Ejemplos con los IDs del servidor. Asigna estos permisos a los grupos de LuckPerms según el rango del jugador.
típicamente por grupo/rango
⭐
Integración con LuckPerms

Asigna frewards.user al grupo user, frewards.ros al grupo ros, etc. El plugin detecta el permiso al abrir la GUI y muestra el estado correcto para cada jugador.

Placeholders

Variables internas

Usa estos placeholders dentro del lore de cada recompensa en rewardsgui.yml.

%cooldown_<id>%
Tiempo restante formateado del cooldown de la recompensa con ese ID. Se actualiza cada vez que el jugador abre la GUI.
▸ ej. %cooldown_ros% → "23h 50m 12s"
%reward_name%
Nombre con color de la recompensa tal como está definido en el campo name de rewardsgui.yml.
▸ ej. %reward_name% → "§aROS" (con color)
%player%
Nombre del jugador que reclama la recompensa. Disponible dentro del campo commands de cada recompensa.
▸ ej. "kit give %player% vip" → "kit give Steve vip"
%rewards%
Número de recompensas disponibles para el jugador. Usado en el mensaje de notificación al conectar (messages.yml).
▸ ej. "Tienes %rewards% recompensas" → "Tienes 4 recompensas"
Configuración

Archivos de configuración

Toda la personalización se realiza mediante archivos YAML. El editor GUI escribe directamente en rewardsgui.yml.

rewardsgui.yml

Define las recompensas, sus estados, lore, comandos, cooldown y slot.

rewardsgui.yml
# Materiales por estado
material-state:
  disponible: "CHEST_MINECART"
  progreso:   "FURNACE_MINECART"
  noperm:    "TNT_MINECART"

gui:
  title: "&#FF9000▸ &#FF4C00Baltiamc Rewards"
  size:  45

rewards:
  ros:
    name:         ".d603&lROS"
    display-name: 'Recompensa %reward_name%'
    permission:   frewards.ros
    cooldown:     86400
    slot:         12
    sound:        "ENTITY_PLAYER_LEVELUP"
    title:        "�FF00¡RECOMPENSA DIARIA!"
    subtitle:     "¡Reclamada con éxito!"
    commands:
      - "crates key give %player% comun 4"
    lore:
      disponible:
        - '&8Descripción'
        - '  &8▪ Estado: &aDisponible ✔'
        - '  &8▪ Cooldown &e24 Horas ⌚'
      progreso:
        - '  &8▪ Estado: &eEn progreso'
        - '  &8▪ Cooldown &c%cooldown_ros%'
      noperm:
        - '  &8▪ Estado: &cBloqueada ✘'
        - '  &8▪ Requieres: %reward_name% ₪'

config.yml

Prefijo y configuraciones globales.

config.yml
prefix: "&#FCD05C&lRECOMPENSAS &7›› "

messages.yml

Todos los mensajes del sistema son configurables.

messages.yml
reward-claimed:  "&a✅ ¡Reclamada con éxito!"
reward-cooldown: "&e⏳ Todavía no puedes reclamar."
no-permission:  "&c⛔ Sin permiso."
reload-success: "&a🔄 Plugin recargado."

join-notify:
  singular: "&fTienes &e%rewards% &frecompensa..."
  plural:   "&fTienes &e%rewards% &frecompensas..."

editor.yml

Personaliza los GUIs del editor: colores, materiales, slots y presets de cooldown.

editor.yml
selector:
  title-prefix: "&#FF9000▸ &#FF4C00Editor"
  filler:       "GRAY_STAINED_GLASS_PANE"

cooldown:
  presets:
    - slot:    10
      name:    "24 Horas"
      seconds: 86400
      material: "CLOCK"
    - slot:    11
      name:    "1 Hora"
      seconds: 3600
      material: "CLOCK"
💾
data.yml — gestionado automáticamente

El plugin guarda los timestamps de cooldown y los ítems físicos de cada recompensa en data.yml. No es necesario editarlo manualmente; el editor GUI lo actualiza automáticamente.

Instalación

Listo en 3 pasos

Instala y configura FRewards en menos de 5 minutos.

1

Coloca el .jar en /plugins

Descarga FRewards.jar y cópialo en la carpeta plugins/ de tu servidor Paper 1.21+. Reinicia para generar los archivos de configuración.

2

Configura rewardsgui.yml

Abre plugins/FRewards/rewardsgui.yml y añade o edita las recompensas: nombre, permiso, cooldown, slot, lore y comandos. O usa el editor GUI con /frewards editor.

3

Asigna permisos y recarga

Asigna frewards.<id> a los grupos de LuckPerms. Ejecuta /frewards reload y los jugadores ya pueden usar /rewards.

Flujo del juego

¿Cómo funciona?

🔔
Notificación al entrar

Al conectarse, el jugador recibe un mensaje automático indicando cuántas recompensas tiene disponibles para reclamar.

🎮
Abre /rewards

El inventario muestra todas las recompensas con su estado actual: disponible (verde), en progreso (naranja) o bloqueada (rojo), según los permisos del jugador.

✅
Reclama y recibe

Al hacer clic en una recompensa disponible, el jugador recibe los ítems físicos y se ejecutan los comandos configurados. Se muestra un título en pantalla y un sonido.

⏳
Espera el cooldown

La recompensa pasa a estado "En progreso" mostrando el tiempo restante. El cooldown se guarda en data.yml y persiste entre reinicios.

REQUERIDO
🔌

OPCIONAL
🔑

LuckPerms

Recomendado para gestionar los permisos por rango (frewards.ros, etc.). Funciona con cualquier plugin de permisos.
